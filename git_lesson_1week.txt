2. 1주차 주요 개념 키워드 적어보기

- Git 이란?
	프로젝트의 버전 관리를 위한 도구이다. 현재 가장 널리 쓰이는 버전 관리의 도구이다.
	
- Github 란?
	Git 원격 저장소 + Git 으로 할 수 있는 커뮤니티 기능 서비스를 말한다.

- Git 을 통한 버전 관리와 Commit 이란?
	Git 에서는 누가, 언제, 현재 프로젝트의 상태가 어떠한지 앞에서 말한 세가지 정보를 포함해 작업내역을 관리
	이렇게 프로젝트 상태를 저장하는 행위를 commit 이라고 표현한다.
	
- commit
	commit 에는 다양한 정보가 담겨 있는데 이는 아래와 같다.
	1. commit id 	: 커밋을 구분하기 위한 유일한 값
	2. 작업 일자 		: 날짜와 시간이 담긴다.
	3. 작업한 사람		: 작성자(author)
	4. 작업 내역 		: 커밋 메시지
	5. 작업 내역의 순서 	: 해당 commit 의 직전 commit 이 무엇인지에 대한 정보
	
- add (스테이징)
	commit 할 때 반영할지 안할지에 대한 선택을 파일 단위로 할 수 있다.
	
- commit history
	내가 계속 commit 했던 것들에 대한 내역이 남아있다.
	
- repo
	Git 으로 관리되는 프로젝트를 repo(repository) 라고 한다. 즉, 약자이다.
	내 컴퓨터에 있는것이 로컬 repo
	원격 공간에 저장되어 있는것을 원격 repo(remote repository) 라고 한다.
	
- tracking(추적)
	로컬 repo 와 원격 repo 를 연결하는 것을 말한다.
	
- pull
	로컬 repo 의 commit 들을 원격 repo 에 넣는 것을 말한다.
	한국말로 하면 밀어넣기 로 이해하면 된다.
	
- push
	원격 repo 의 commit 들을 로컬 repo 로 반영하는 것을 말한다.
	이 또한 한국말로 하면 당겨오기 로 이해하면 된다.
	
- clone
	원격 repo 를 내 컴퓨터에 가져와서 초기 repo 셋팅 하는것을 말한다.
	무슨 말이냐면 내 컴퓨터에 원격 repo 의 내용이 없을 때 가져오는 즉, 복제하는 것을 말한다.
